# code-study 스킬

사용자가 제공한 코드 또는 개념을 분석해 아래 형식의 학습 자료를 생성합니다.

## 순서

### 1단계: 대상 파악

- 사용자가 코드를 선택(IDE selection)했으면 그 코드를 대상으로 합니다.
- 파일명이나 개념명만 언급했으면 해당 파일을 읽어서 분석합니다.
- 대상이 불명확하면 "어떤 코드나 개념에 대한 학습 자료를 만들까요?" 라고 물어봅니다.

### 2단계: 저장 경로 결정

학습 자료를 저장할 md 파일 경로를 결정합니다.

- 현재 작업 중인 프로젝트 루트에 `docs/study/` 디렉토리가 없으면 생성합니다.
- 파일명은 분석 대상 이름을 기반으로 합니다. 예: `canvas-api.md`, `use-ref-pattern.md`
- 사용자가 경로를 지정했으면 그 경로를 사용합니다.

### 3단계: 학습 자료 작성 및 파일 저장

아래 형식으로 내용을 작성한 뒤 **Write 도구로 md 파일에 저장**합니다.
저장 완료 후 사용자에게 파일 경로를 알려줍니다.

## 작성 가이드

**🔍 한눈에 보기**
- 이 코드가 "어떤 문제를 어떻게 해결하는가"를 비개발자도 이해할 수 있는 수준으로 씁니다.
- 주요 개념 목록은 실제로 코드에서 쓰인 것만 나열합니다.

**🪜 Step by Step**
- 코드의 실행 흐름 순서대로 나눕니다.
- 각 Step은 하나의 책임/역할 단위로 분리합니다.
- 코드 스니펫은 전체가 아닌 해당 Step의 핵심 부분만 발췌합니다.
- 💡 팁은 해당 Step에서 자주 실수하거나 놓치는 부분을 씁니다.

**🧠 개념 정리**
- 개념 수는 3~6개가 적당합니다. 너무 기초적인 것(변수, if문 등)은 제외합니다.
- 예제는 이 코드와 무관한 독립적인 예제여야 합니다. 학습자가 개념을 범용적으로 이해할 수 있어야 합니다.
- "이 코드에서의 역할"은 추상적 개념과 실제 코드를 연결하는 다리 역할입니다.
- "어떤 기능에 많이 활용되나요?"는 실무에서 자주 보이는 패턴 위주로 씁니다.

**📝 정리**
- 단순 요약이 아닌, 학습자가 다음번에 비슷한 코드를 짤 때 기억해야 할 핵심 원칙을 씁니다.
- 5개 내외로 유지합니다.

## 주의사항

- 결과물은 반드시 md 파일로 저장합니다. 채팅창에만 출력하고 끝내면 안 됩니다.
- 저장 전에 `docs/study/` 디렉토리가 없으면 먼저 생성합니다.
- 코드 블록 언어는 실제 코드 언어에 맞게 지정합니다 (ts, tsx, js, python 등).
- 개념 설명은 해당 언어/프레임워크에 맞는 맥락으로 작성합니다.
- 학습자 수준은 해당 언어는 알지만 이 개념은 처음 접하는 중급자로 가정합니다.
